HTML docs

Elements:

<elementName>Content goes here</elementName>


Void elements, no closing tag:

<img>
<meta>


Attributes, placed in the opening element, provides additional info:

<elementName attribute="value"></elementName>


Boolean attributes are either present or absent in an HTML tag. If present, value is true, otherwise false

examples: disabled, readonly, required


Comments:

<!-- Comment -->


Common elements:

Headers
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>

Paragraph
<p>This is a paragraph element</p>


Image
<img src="link/file path here">

add alt attribute incase image doesnt load


Body, represents the content for the HTML document
<body>
  <h1>Header</h1>
  <p>This is a paragraph element</p>
</body>


Section, divide the content into smaller sections
<section>
  <h2>Wassup</h2>
  <p>Hi, I am a web developer.</p>
</section>


Division, generic container to hold elements
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>


Anchor, used to apply links to the webpage. href attribute specifies where the link should go when clicked
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>


Unordered list (bullet list)
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>

Ordered list (numbered)
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>


Emphasis (sideways)
<p>I <em>love</em> pizza</p>


Strong emphasis (bold)
<p>This is <strong>very</strong> good</p>


Footer, placed at the bottom, usually cointains copyright info and other important links
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>


ID and Grouping:

IDs, unique element identifiers, used once per document. Cannot contain spaces, 
<h1 id="title">Movie Review Page</h1>
<div id="red-box"></div>

Classes, also element identifiers but can be used on multiple elements, can contain spaces
<div class="box"></div>

<div class="box red-box"></div>
<div class="box blue-box"></div>


Link, used to link external resources like stylesheets and site icons
<link rel="stylesheet" href="./styles.css" />


Script, used to embed executable code
<script>
    alert("Welcome to freeCodeCamp");
</script>

Target attributes
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>

_self opens in the current window/tab
_blank opens a new window/tab
_parent opens in the parent of the current context
_top opens in the top-most browsing context


Link states:
:link (untouched)
:visited (visited the page that the link links to)
:hover (when a user is hovering on a link)
:focus (when we focuses on the link)

Math Equations and Chemical Formulas:

Superscript, displays a piece of text printed above the normal line of text
<p>2<sup>2</sup> (2 squared) is 4.</p>

<p>
  Monseigneur is often written as <strong>M<sup>gr</sup></strong>.
</p>


Subscript, same as superscript but lower than baseline for text
<p>CO<sub>2</sub></p>


Code in text, code only represents single line code, so if more lines wrap code in pre
<p>
  To set the text color to blue in CSS, use the following code:
  <code>color: blue;</code>
</p>

Preformatted text
<pre>
  <code>
    body {
      color: red;
    }
  </code>
</pre>



Structural hierarchy:

Header element, used to define the header of a document or section
<header>
  <h1>CatPhotoApp</h1>
  <p>Welcome to our cat gallery.</p>
</header>


Main element, contain the main content of the web page
<main>
  <section>
    <h2>Cat Photos</h2>
    <p>Browse adorable cat pictures.</p>
  </section>
</main>


Section element, divide content into smaller sections
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>


Navigation Section, section with navigation links, list not neccessary, but better
<nav>
  <ul>
    <li><a href="#photos">Photos</a></li>
    <li><a href="#videos">Videos</a></li>
  </ul>
</nav>


Figure/figcaption, figure groups content like images an diagrams, figcaptions represent a caption for that content
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>  
</figure>


Idiomatic text, highlights alternative voice or moods etc.
<p>
  There is a certain <i lang="fr">je ne sais quoi</i> in the air.
</p>


Unarticulated Annotation, represents a span of inline text that is rendered non textual annotation
<p>
  You can use the unarticulated annotation element to highlight
  <u>inccccort</u> <u>spling</u> <u>issses</u>.
</p>


Ruby Annotation (ruby), used for text with pronounciation or meaning explanations
Ruby fallback parenthesis (rp)m used as a fallback for browsers lacking support
Ruby text (rt), indicate text for the ruby annotation, used for pronounciation
<ruby>
  明日 <rp>(</rp><rt>Ashita</rt><rp>)</rp>
</ruby>


Strikethrough element, used to represent content no longer accurate
<p>
  <s>Tomorrow's hike will be meeting at noon.</s>
</p>
<p>
  Due to unforeseen weather conditions, the hike has been canceled.
</p>


Internal links, used to link another section of the page, skipping links table of contents
<a href="#about-section">Go to About Section</a>

<section id="about-section">
  <h2>About</h2>
  <p>This is the about section of the page.</p>
</section>
